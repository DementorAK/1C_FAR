Мультиплатформенный плагин для **FAR Manager** / **far2l** для прозрачной работы с бинарными артефактами **1С:Предприятие 8** — как с обычными архивами.

## Поддерживаемые форматы

| Формат | Описание | Чтение | Запись |
|--------|----------|--------|--------|
| `.epf` | Внешняя обработка | ✓ | ✓ |
| `.erf` | Внешний отчёт | ✓ | ✓ |
| `.cf`  | Конфигурация | ✓ | ✓ |
| `.cfe` | Расширение конфигурации | ✓ | ✓ |
| `.1cd` | Файловая база данных | 🔜 | 🔜 |

## Установка

### Сборка из исходников

**Требования:** [Rust toolchain](https://rustup.rs/) (1.88+)

```bash
cargo build --release
```
*(Примечание: сборка по умолчанию настроена под Windows / FAR 3)*

### Windows (FAR Manager 3)

1. Собрать плагин (см. выше)
2. Создать каталог плагина: `%FARHOME%\Plugins\far1c\`
3. Скопировать `target\release\far1c.dll` в каталог плагина
4. Скопировать языковые файлы `far1c_en.lng` и `far1c_ru.lng` в тот же каталог
5. Перезапустить FAR Manager
6. Плагин появится в меню `F11`

### Linux (far2l)

Самый простой способ установки в Linux — использовать скрипт установки, который собирает плагин и копирует его в нужные FHS-каталоги системы.

1. Сделайте скрипт исполняемым и запустите его:
   ```bash
   chmod +x install_linux.sh
   ./install_linux.sh
   ```
2. Перезапустите far2l

> **Важно:** Скрипт использует `sudo` для установки бинарного файла плагина в `/usr/lib/far2l/Plugins/far1c/far1c.far-plug-wide` и файлов локализации в `/usr/share/far2l/Plugins/far1c/plug/` в соответствии со стандартом FHS far2l. Языковые файлы необходимы для корректной локализации интерфейса плагина.

> **Важно:** Языковые файлы (`*.lng`) необходимы для корректной локализации интерфейса плагина. Без них вместо текста будут отображаться идентификаторы сообщений.

## Использование

### Открытие артефакта

```
Enter / Ctrl+PgDn на .epf/.cf/.cfe  →  Открывает виртуальную панель внутри артефакта
Backspace / Ctrl+PgUp               →  Выход из виртуальной панели
```

Плагин также можно вызвать из меню `F11` — он откроет текущий выбранный файл.

### Операции внутри виртуальной панели

| Клавиша | Действие |
|---------|----------|
| `F3` | Просмотр файла (viewer) |
| `F4` | Редактирование файла (editor) — изменения сохраняются обратно в контейнер |
| `F5` | Копирование из артефакта на диск |
| `F6` | Копирование с диска в артефакт |

### Настройки

Доступ к настройкам: `F11` → Plugin Configuration, или `F9` → Options → Plugins configuration.

Доступные опции:

- **Создавать бэкап перед сохранением** — создаёт резервную копию с временной меткой (например, `test.20260512-143022.epf`)
- **Стиль распаковки** — Raw / Full-parse / V8Unpack-style / Saby-style

## Архитектура

Проект построен на трёхслойной архитектуре:

```
src/
├── lib.rs                       # Точка входа, условный экспорт Far Plugin API
├── far/                         # СЛОЙ 1: Взаимодействие с Far Manager (Dual-API)
│   ├── far3/                    # Имплементация для FAR 3 (Windows)
│   │   ├── api.rs               # Биндинги Far Plugin SDK 3.0
│   │   └── exports.rs           # Экспортируемые C ABI функции
│   ├── far2/                    # Имплементация для far2l/far2m (Linux/macOS)
│   │   ├── api.rs               # Биндинги far2l Plugin API
│   │   └── exports.rs           # Экспортируемые C ABI функции
│   ├── string_utils.rs          # Кроссплатформенные утилиты для строк (u16/u32)
│   ├── traits.rs                # Трейт FarHost как абстракция над API
│   ├── panels.rs                # Логика виртуальных файловых панелей (VFS, навигация, commit)
│   ├── ui.rs                    # Элементы UI (диалоги, прогресс-бары, меню)
│   ├── lang.rs                  # Локализация через .lng-файлы
│   └── settings.rs              # Настройки плагина (стиль распаковки, бэкап)
├── v8/                          # СЛОЙ 2: Семантика артефактов 1С
│   ├── container.rs             # Чтение CF-контейнеров (ImageHeader, rows, страницы)
│   ├── vfs_builder.rs           # Построение VFS-дерева из рядов контейнера
│   ├── writer.rs                # CF-writer: пересборка контейнеров
│   ├── uuids.rs                 # UUID-ы типов объектов метаданных 1С
│   └── tests.rs                 # Интеграционные тесты
└── base/                        # СЛОЙ 3: Низкоуровневые примитивы (I/O, парсинг)
    ├── reader.rs                # Абстрактный ридер (FileReader, StringReader)
    ├── parser.rs                # Парсер скобочного формата метаданных 1С
    └── deflate.rs               # DEFLATE через flate2
```
