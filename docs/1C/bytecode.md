# Байт-код 1С:Предприятие 8

## 1. Обзор

Байт-код 1С — это скомпилированное представление модуля 1С, которое исполняется виртуальной машиной платформы. Используется в защищённых модулях, где исходный текст BSL недоступен.

### 1.1 Когда встречается

- Защищённые модули конфигураций
- Внешние обработки/отчёты с закрытым кодом
- Модули, сохранённые без исходного текста

### 1.2 Файлы в контейнере

Внутри CF/EPF/ERF защищённый модуль представлен файлами:

| Файл | Описание |
|------|----------|
| `text` | Исходный код (если доступен) или пустой |
| `image` | Скомпилированный байт-код |
| `info` | Метаданные модуля (версия, флаги) |

### 1.3 Структура файла image

```
image
├── Version (4 байта)           -- Версия формата
├── Header                      -- Заголовок
│   ├── CodeSize (4 байта)     -- Размер кода
│   ├── ConstSize (4 байта)    -- Размер констант
│   ├── VarSize (4 байта)      -- Размер переменных
│   └── ProcSize (4 байта)     -- Размер процедур
├── Code                        -- Код программы
│   └── Commands[]              -- Массив команд (OpCode + Argument)
├── Const                       -- Таблица констант
│   └── Values[]               -- Значения (числа, строки, даты)
├── Var                        -- Таблица переменных
│   └── Names[]                -- Имена переменных
└── Proc                       -- Таблица процедур/функций
    └── Entries[]              -- Точки входа процедур
```

## 3. Формат команды

Каждая команда байт-кода состоит из:

```
[OpCode] [Argument]
```

- **OpCode** — 1 байт (0-255)
- **Argument** — переменная длина:
  - Для большинства команд — 4 байта (число)
  - Для переходов — смещение или адрес

### 3.1 Особые команды

| OpCode | Argument | Описание |
|--------|----------|----------|
| ArgNum | 0-255 | Количество параметров для следующего вызова |
| CallProc | индекс | Индекс в таблице констант |
| CallFunc | индекс | Индекс в таблице констант |
| CallLoc | индекс | Индекс в таблице процедур |
| Jmp | адрес | Абсолютный адрес |
| JmpFalse | адрес | Условный переход |
| LdConst | индекс | Индекс в таблице констант |

## 4. Дизассемблер

### 4.1 Алгоритм дизассемблирования

```
1. Прочитать заголовок image
2. Вычислить смещения секций
3. Загрузить таблицу констант
4. Загрузить таблицу переменных
5. Загрузить таблицу процедур
6. Для каждой команды в Code:
   a. Прочитать OpCode
   b. Если есть аргумент — прочитать
   c. Декодировать в мнемонику
   d. Добавить в листинг
```

### 4.2 Пример вывода дизассемблера


## 5. Ассемблер (обратная сборка)

### 5.1 Алгоритм сборки

```
1. Парсить текстовый листинг
2. Построить таблицы: константы, переменные, процедуры
3. Для каждой команды:
   a. Найти/добавить в таблицу
   b. Закодировать OpCode + Argument
4. Записать заголовок
5. Записать секции (Code, Const, Var, Proc)
```

### 5.2 Ограничения

- Синтаксис ассемблера не стандартизирован
- Восстановление из байт-кода ≠ оригинальный исходный код
- Не гарантируется полная совместимость

## 6. Применение в far1c

### 6.1 FR-061: Просмотр защищённых модулей

```
Задачи:
1. Обнаружить файл image в контейнере
2. Распознать защищённый модуль (нет text или text пустой)
3. Дизассемблировать image
4. Показать листинг в viewer
```

### 6.2 FR-062: Редактирование (опционально)

```
Задачи:
1. Принять изменения из отредактированного листинга
2. Ассемблировать обратно в image
3. Обновить контейнер
4. Валидировать байт-код
```

### 6.3 Индикация в панели

```
Защищённые модули отображаются с суффиксом:
  [protected]

Или специальным значком в колонке типа.
```
