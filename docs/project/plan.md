# План инициализации проекта: 1C Far Plugin

## Обзор

Документ описывает план подготовки и реализации мультиплатформенного плагина для FAR Manager / far2l, обеспечивающего прозрачную работу с бинарными артефактами платформы 1С:Предприятие 8.

**Язык реализации:** Rust с C-совместимым интерфейсом (C ABI) для Far Plugin API

**Целевые платформы:** Windows (FAR Manager 3), Linux (far2l)

**Основные источники:**

- docs/cf_processing.md — внутренняя спецификация алгоритмов парсинга CF
- tool1cd (GitHub e8tools) — формат .1cd
- v8asm (GitHub EvilBeaver) — справочник по байт-коду

### Отклонение структуры от плана

Плановая архитектура (согласно ADR-002) была успешно перенесена в рабочую директорию. Теперь структура состоит из 3 логических слоев:

1. `src/far/` (Взаимодействие с Far Manager)
2. `src/v8_artifacts/` (Бизнес-логика форматов 1С)
3. `src/base/` (Низкоуровневые I/O операции)

Тестовые артефакты для завершения Фазы 2 и старта Фазы 3 уже подготовлены, что опережает первоначальный план для Фазы 0.

Однако, в рабочей папке пока есть только пустые директории слоев и пустой `lib.rs`. Среда разработки Rust не инициализирована (нет Cargo), и Git-репозиторий не создан.

## Стратегия DEFLATE

Для распаковки сжатых блоков CF-контейнеров используется Rust crate `flate2` — нативная реализация DEFLATE без внешних зависимостей.

| Компонент | Решение | Примечание |
|-----------|---------|------------|
| DEFLATE-декомпрессия | `flate2` (crate) | Нативная производительность, кроссплатформенность |
| DEFLATE-компрессия | `flate2` (crate) | Для CF-writer (пересборка контейнеров) |

## Архитектура модулей (из SRS v2.0)

```text
src/
├── lib.rs                       -- Точка входа, экспорт Far Plugin API (C ABI)
├── far/                         -- СЛОЙ 1: Взаимодействие с Far Manager
│   ├── mod.rs                   -- Инициализация, настройки, диспетчеризация вызовов
│   ├── panels.rs                -- Логика виртуальных файловых панелей (VFS для артефактов)
│   └── ui.rs                    -- Отрисовка элементов UI (диалоги, прогресс-бары, меню)
├── v8_artifacts/                -- СЛОЙ 2: Семантика и структура артефактов 1С
│   ├── mod.rs
│   ├── cf.rs                    -- Высокоуровневая обработка контейнеров CF/EPF/ERF
│   ├── metadata.rs              -- Навигация и разбор дерева метаданных конфигурации
│   ├── onecd.rs                 -- Логика парсинга внутреннего формата файловых баз .1cd
│   └── bytecode.rs              -- Байт-код: дизассемблер и структуры (opcodes)
└── base/                        -- СЛОЙ 3: Низкоуровневые библиотечные функции (I/O, парсинг)
    ├── mod.rs
    ├── reader.rs                -- Абстрактный ридер (буферизованное чтение, работа с диском/памятью)
    ├── parser.rs                -- Низкоуровневый парсер скобочных структур 1С
    └── utils.rs                 -- Утилиты (нормализация путей, кодировки, временные файлы, deflate)
```

### Задачи Фазы 1 (детализация)

**1.1 Cargo-проект + Far Plugin API**

- Создать `Cargo.toml` с `crate-type = ["cdylib"]`
- Реализовать `lib.rs` с `extern "C"` экспортом: SetStartupInfoW, GetPluginInfoW, OpenW, AnalyseW
- Зарегистрировать префикс "1c:"
- Обработать расширения: .epf, .erf, .cf, .cfe, .1cd
- Тест: плагин виден в F11 → Plugins

**1.2 GetFindDataW — базовое сканирование**

- Реализовать структуру PluginPanelItem
- Читать ImageHeader через CF-парсер
- Возвращать список файлов/папок
- Тест: при входе в артефакт панель не пустая

**1.3 SetDirectoryW — навигация**

- Обрабатывать ".." для возврата
- Обрабатывать SetDirectoryW(Dir)
- Обновлять заголовок панели
- Тест: Backspace возвращает на уровень выше

**1.4 CF-парсер на Rust**

- Портировать алгоритмы FileReader/StringReader из [docs/cf_processing.md](docs/1C/cf_processing.md)
- Реализовать итератор Image::rows()
- Тест: чтение структуры .cf файла

**1.5 DEFLATE через flate2**

- Подключить crate `flate2`
- Реализовать `utils/deflate.rs`
- Тест: распаковка сжатых блоков работает
